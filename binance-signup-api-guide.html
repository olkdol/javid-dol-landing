How to Create a Binance Account & Futures API Key

Everything you need before your first trade with JaviD Future Bot — account signup, identity verification, enabling Futures, and generating an API key with the correct permissions.

Written for the current Binance web flow · Last checked July 2026

On this page

  1. Create your Binance account
  2. Complete identity verification (KYC)
  3. Enable your Futures account
  4. Create your API key
  5. Set the correct permissions (the step people miss)
  6. Connect it to JaviD Future Bot
  7. Troubleshooting & FAQ
1

Create Your Binance Account

Registration takes about 5 minutes. Use our official partner signup link below — it's the same Binance registration flow, at zero extra cost to you, and it's how JaviD Future Bot stays free with no subscription fee.

Don't have an account yet? Sign up through our partner link — same official Binance flow, no extra cost. Create Account →
  • Go to binance.com and click Register.You can sign up with an email address, a phone number, or continue with your Google/Apple account.
  • Set your password.Minimum 8 characters with at least one uppercase letter and one number — we'd recommend adding a symbol too for a stronger key.
  • Enter the 6-digit verification code sent to your email or phone.The code is time-limited (typically ~30 minutes), so check your inbox/spam folder right away.
  • Turn on Two-Factor Authentication (2FA) immediately from your account security settings.Google Authenticator or a Passkey both work. You'll need 2FA active later to create an API key.
Tip: Use a password manager and a dedicated email for exchange accounts — it's the single easiest way to avoid account-takeover phishing.
2

Complete Identity Verification (KYC)

Registering alone doesn't unlock trading. You need identity verification before you can deposit meaningfully or trade Futures.

  • Open your profile menu → Identification.Select your country/region of residence and the ID document type (national ID card, driver's license, or passport).
  • Upload photos of your ID, then complete the facial verification (a quick selfie/liveness check).
  • Wait for approval — this usually clears in 5–15 minutes. Your profile status changes from Unverified to Verified once approved.
Note: Some higher-tier API permissions now require the "Verified Plus" identity level rather than basic verification. If your API Management page shows restrictions you can't clear, check whether your account needs the additional verification step.
3

Enable Your Futures Account — Before You Create an API Key

This step is easy to do out of order, and doing it out of order is the single most common setup mistake.

  • Open the [Futures] menu from the top navigation.
  • Complete the futures risk questionnaire if this is your first time — a short quiz confirming you understand leveraged trading risk.
  • Activate your USDⓈ-M Futures account.Once active, you'll see a Futures wallet alongside your Spot wallet.
Do this before Part 4: if your API key already existed before you activated Futures (or if you have Portfolio Margin enabled), that key will never be able to get Futures permission — you'll have to delete it and create a fresh one. Enable Futures first, API key second.
4

Create Your API Key

Before you can create a key, Binance requires: 2FA enabled, at least a token deposit into your Spot wallet (to activate the account), and completed identity verification — all covered in Parts 1–2.

  • Profile icon → Account → API Management.
  • Click Create API, then choose System generated.This uses standard HMAC authentication and gives you an API Key + Secret Key — the pairing JaviD Future Bot expects.
  • Give it a label, e.g. JaviD-Bot, so you can recognize it later.
  • Verify with 2FA (authenticator code, passkey, or email confirmation).
  • Your key is created — but by default it can only read data. Continue to Part 5 before using it.
5

Set the Correct Permissions

A freshly created key defaults to Enable Reading only. To let JaviD Future Bot place orders, you need to edit the key:

  • Check "Enable Futures."
  • Add an IP restriction — this is mandatory, not optional.Since a January 2023 policy change, Binance will not grant any permission beyond read-only unless the key is restricted to specific IPs. Choose "Restrict access to trusted IPs only" and enter the public IPv4 address of the PC that will run the bot (search "what is my ip", or use JaviD's built-in "Run Connection Diagnostics" button, which detects and reports it for you).
  • Leave "Enable Withdrawals" OFF.A trading bot never needs withdrawal permission. If anything you're configuring asks you to turn this on, stop and reconsider.
  • Save, then wait 5–10 minutes for the IP whitelist change to fully propagate before testing the connection.
Copy your Secret Key now: it is shown exactly once. If you navigate away without saving it, you'll need to delete the key and generate a new one.
6

Connect It to JaviD Future Bot

  • Launch JaviD Future Bot (start.bat) — your browser opens automatically to the dashboard.
  • Go to the Settings tab and paste your Access Key and Secret Key.Watch for accidental leading/trailing spaces when pasting.
  • Click Save, then Test Connection. A green checkmark means you're ready to go.
  • Turn on Shadow Mode and verify the bot's behavior for a few days before ever switching to live trading.

Troubleshooting & FAQ

SymptomCause / Fix
HTTP 451 errorBinance blocks Futures API access from some regions, including Korean IPs. Run the bot from an overseas VPS or through a VPN.
-2015 error on Test ConnectionYour IP restriction hasn't propagated yet, or you entered the wrong IP. Wait 5–10 minutes and re-check the address.
-1022 Signature errorAlmost always a Secret Key copy/paste issue — re-copy it, watching for hidden line breaks or spaces.
-1021 Timestamp errorYour PC's clock has drifted. Turn on automatic time sync in Windows settings.
Can't enable Futures permission on the keyThe key was created before your Futures account was activated (see Part 3). Delete it and create a new one, in the right order.
Can't enable any permission beyond ReadYou skipped the IP restriction step in Part 5 — it's mandatory for any non-read permission.
