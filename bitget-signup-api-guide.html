How to Create a Bitget Account & Futures API Key

Everything you need before your first trade with JaviD Future Bot on Bitget — account signup, identity verification, enabling Futures, and generating an API key with the correct permissions.

Written for the current Bitget web flow · Last checked July 2026

On this page

  1. Create your Bitget account
  2. Complete identity verification (KYC)
  3. Enable your USDT-M Futures account
  4. Create your API key
  5. Set the correct permissions (the step people miss)
  6. Connect it to JaviD Future Bot
  7. Keep it running 24/7 (cloud VM or mini PC)
  8. Troubleshooting & FAQ
1

Create Your Bitget Account

Registration takes about 5 minutes. Use our official partner signup link below — it's the same Bitget registration flow, at zero extra cost to you, and it's how JaviD Future Bot stays free with no subscription fee.

Don't have an account yet? Sign up through our partner link — same official Bitget flow, no extra cost. Create Account →
  • Open the partner link above and click Sign Up.You can register with an email address, a phone number, or continue with your Google/Apple account.
  • Set your password.Use a strong, unique password — we'd recommend a password manager and adding a symbol for a stronger key.
  • Enter the verification code sent to your email or phone.The code is time-limited, so check your inbox/spam folder right away.
  • Turn on Two-Factor Authentication (2FA) immediately from your account security settings, and set a security/API passphrase you'll remember.Google Authenticator or a Passkey both work. You'll need 2FA active later to create an API key, and Bitget requires a separate Passphrase for every API key.
Tip: Use a password manager and a dedicated email for exchange accounts — it's the single easiest way to avoid account-takeover phishing.
2

Complete Identity Verification (KYC)

Registering alone doesn't unlock trading. You need identity verification before you can deposit meaningfully or trade Futures.

  • Open your profile menu → Identity Verification.Select your country/region of residence and the ID document type (national ID card, driver's license, or passport).
  • Upload photos of your ID, then complete the facial verification (a quick selfie/liveness check).
  • Wait for approval — this usually clears in a few minutes to a few hours. Your profile status changes to Verified once approved.
Note: Some regions require an extra address/proof-of-residence step for higher trading or withdrawal limits. Check your account's verification level if you hit a limit later.
3

Enable Your USDT-M Futures Account — Before You Create an API Key

This step is easy to do out of order, and doing it out of order is the single most common setup mistake.

  • Open the [Futures] menu from the top navigation.
  • Complete the futures risk questionnaire if this is your first time — a short quiz confirming you understand leveraged trading risk.
  • Activate your USDT-M Futures account.Once active, you'll see a Futures wallet alongside your Spot wallet, and you'll need to transfer some USDT into it to trade.
Do this before Part 4: enable Futures first, API key second, so the key can be granted futures permission from the start.
4

Create Your API Key

Before you can create a key, Bitget requires 2FA (or phone verification) enabled and completed identity verification — both covered in Parts 1–2.

  • Profile icon → API Management.
  • Click Create API Key, then choose System-generated.This gives you an API Key + Secret Key + Passphrase — Bitget is one of the exchanges that requires all three, unlike some others that only use a Key/Secret pair.
  • Give it a name, e.g. JaviD-Bot, so you can recognize it later.
  • Set your own Passphrase and write it down — you choose this yourself, and JaviD Future Bot will need it alongside the Key and Secret.
  • Verify with 2FA (authenticator code or email confirmation) to finish creating the key.
5

Set the Correct Permissions

When configuring the key, grant only what JaviD Future Bot actually needs to trade — nothing more.

✅ Check exactly these, nothing else:
  • Read (usually on by default)
  • Spot Trade / Orders
  • Futures Trade / Futures-Holdings
❌ Leave withdrawal permission unchecked: a trading bot never needs it. Leaving it off means that even in a worst-case scenario where your API key was ever exposed, no one could move funds off the exchange — your assets stay yours.
  • Add an IP restriction if the option is offered on your account.Choose "Restrict access to trusted IPs only" and enter the public IPv4 address of the machine that will run the bot (search "what is my ip", or use JaviD's built-in "Run Connection Diagnostics" button once the Bitget build supports it).
  • Leave withdrawal permission OFF.If anything you're configuring asks you to turn this on, stop and reconsider.
  • Save, then allow a few minutes for any IP whitelist change to propagate before testing the connection.
Copy your Secret Key and Passphrase now: the Secret Key is shown exactly once. If you navigate away without saving it, you'll need to delete the key and generate a new one.
6

Connect It to JaviD Future Bot

  • Launch JaviD Future Bot (start.bat) — your browser opens automatically to the dashboard.
  • Go to the Settings tab and select Bitget, then paste your API Key, Secret Key, and Passphrase.Watch for accidental leading/trailing spaces when pasting — this is the most common connection error with Bitget specifically, since it needs all three fields correct.
  • Click Save, then Test Connection. A green checkmark means you're ready to go.
  • Turn on Shadow Mode and verify the bot's behavior for a few days before ever switching to live trading.
7

Keep It Running 24/7

JaviD Future Bot only trades while it's actually running — the futures market never closes, so the machine running the bot needs to stay powered on and connected 24 hours a day, 365 days a year. Your everyday laptop works for a first test, but most users move to one of these two setups once they go live:

  • A cloud virtual machine (Amazon AWS EC2, or any comparable VPS provider).Rented by the month, always on, remotely accessible from anywhere. A small/low-tier instance is enough — the bot itself is lightweight.
  • A dedicated mini PC or spare desktop, left on at home.An Intel NUC-class mini PC (or any always-on machine you don't otherwise need) draws very little power and can be plugged into a UPS to survive short outages. Make sure Windows sleep/hibernate is disabled and the internet connection is stable.
Either option works equally well. The only requirements are: the machine stays powered on, stays connected to the internet, and (per Part 5) has its IP address whitelisted if you set that restriction. If you move the bot to a new machine or your IP changes, update the IP restriction on the key.
Reminder: if the machine restarts or loses power while a position is open, close that position manually from the Bitget app under Futures before restarting the bot — see the User Manual for details.

Troubleshooting & FAQ

SymptomCause / Fix
Signature/auth errorAlmost always a Secret Key or Passphrase copy/paste issue — re-copy both, watching for hidden line breaks or spaces.
Timestamp errorYour PC's clock has drifted. Turn on automatic time sync in Windows settings.
Can't enable Futures permission on the keyThe key was created before your Futures account was activated (see Part 3). Delete it and create a new one, in the right order.
Orders rejected / insufficient balanceMake sure USDT has been transferred from your Spot wallet into your Futures wallet — they're separate balances.
